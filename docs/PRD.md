## 赛博神父 (Project Convergence) MVP 版本 PRD

### 1. 产品愿景与价值观 (Product Vision)

* **产品定位：** 一款结合 LLM 语义理解与 SCM（结构因果模型）内核的“反事实”情绪抚慰 Agent。
* **核心使命：** 用严谨的数学逻辑与无情的反事实推演，击碎用户对于“平行宇宙中完美自我”的幻想。证明在绝大多数情况下，受限于个体的隐变量（性格、认知、风险偏好）和宏观环境的混淆因子，命运的世界线最终都会发生收束。
* **核心价值观：** 过去无法改变，沉溺于“如果当初”毫无意义。认清现实的必然性，放下执念，积极展望未来。

### 2. 目标受众 (Target Audience)

* **深夜情绪内耗者：** 沉浸在过往感情、升学、人生重大抉择遗憾中的人。
* **交易/投资复盘者：** 经常产生“如果我当时没卖/买就好了”这种 Hindsight Bias（后见之明偏误）的群体。
* **泛二次元与极客群体：** 对“世界线”、“平行宇宙”、“因果律”等概念感兴趣，愿意为优秀的创意买单的用户。

### 3. 核心功能与用户故事 (User Stories)

* **故事倾诉 (The Confession)：** 用户可以输入一段非结构化的自然语言，讲述一段自己深感后悔的往事，并提出核心的“反事实疑问”（例如：“如果我当时没去那个大厂，是不是现在就不会生病了？”）。
* **赛博神父的推演 (The Computation)：** 系统在后台将用户的自然语言解构成因果图（DAG），设定隐变量分布，并运行反事实引擎。
* **神父的判词 (The Verdict)：** 系统返回一段充满神父 Persona 的回复，告知用户其设想的“另一条世界线”的最终收敛结局，通常会无情地指出：即使改变了当初的决定，由于其他底层变量的作用，结局依然不会更好，甚至更糟。

### 4. 系统交互流程 (Agent Workflow)

1. **输入层：** 接收用户文本。
2. **认知层 (LLM Controller)：** 解析文本，提取 Treatment (实际决定), Outcome (现状), Confounders (环境因素), Latent Variables (个人性格等隐变量)。
3. **翻译层 (JSON Contract)：** 将认知层提取的信息转化为标准化的 JSON 格式，定义节点关系和先验概率分布。
4. **计算层 (SCM Simulator)：** 轻量级 Python 后端接收 JSON，执行 `do(X)` 干预，进行蒙特卡洛采样或贝叶斯网络推演，计算反事实概率。
5. **输出层 (LLM Persona)：** 将冰冷的概率数据转化为“赛博神父”的毒舌/悲悯判词返回给用户。

### 5. 界面与 UI/UX 规范 (UI/UX Guidelines)

* **视觉风格：** 极简主义、冷色调、带有一点赛博朋克或宗教建筑的静谧感（比如暗黑背景配上微弱的霓虹十字架或矩阵代码）。
* **交互体验：**
* 对话框式交互，模拟告解室。
* 在后台引擎计算时（因果推演过程中），界面可以展示一些跳动的 Mermaid 因果图或公式，增加“硬核计算”的仪式感。


* **Footer 核心宣告 (强制要求)：** 在所有界面的最下方，必须以优雅且醒目的字体常驻展示产品的初衷文案：
> *"人总是经常后悔，总是会回想若是当初做的决定不一样，现在是什么样子。但是无论如何，过去无法改变，我们应当避免沉溺过去，而是积极展望未来。赛博神父借助因果推断的思想，帮你探索另一种情况的平行宇宙世界线是否发生了跃迁，亦或是收束到了同个结局。愿你放下执念，大步向前。"*



### 6. MVP 演进路线 (Roadmap)

* **Phase 1 (MVP)：** 纯文本交互，基于 LLM 的常识推断生成轻量级 JSON，配合基础概率计算器，主打“世界线收束”的情绪价值与传播度。
* **Phase 2 (垂直深化)：** 针对交易、投资等具有清晰历史数据的场景，引入真实的宏观与市场数据 API，做严肃的 SCM 建模，提供不仅有趣而且极其硬核的回测暴击。
